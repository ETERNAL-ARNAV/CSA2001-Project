# SMS Spam Detection System with Active Learning

An end-to-end Machine Learning pipeline designed to classify SMS messages as "Spam" or "Ham" (legitimate). This project implements a custom Natural Language Processing (NLP) workflow and a self-improving feedback loop.

## 🚀 How It Works

### 1. Data Preprocessing (The "Cleaning" Phase)
Raw text is messy. Before the AI can understand it, we pass every message through a pipeline in `src/preprocess.py`:
- **Lowercase Conversion**: Standardizes "FREE" and "free" as the same word.
- **Punctuation Stripping**: Removes noise like `!` and `$` so the AI focuses on the core vocabulary.
- **Stemming (Porter Stemmer)**: Reduces words to their root form (e.g., "Winner" and "Winning" both become "win").
- **Stopword Removal**: Filters out useless words like "is", "the", and "a" that don't help in identifying spam.

### 2. The Vectorization (Words to Numbers)
Computers can't read text, they only read math. We use `CountVectorizer` to turn our cleaned sentences into a **Sparse Matrix**. Each row represents a message, and each column represents a word count from our vocabulary.

### 3. The Model (Multinomial Naive Bayes)
We chose the **Multinomial Naive Bayes** algorithm. It is the industry standard for document classification because it calculates the probability of a label based on the frequency of specific "Spammy" words. Our current model achieves a **97.76% accuracy** on the test set.

### 4. Active Learning Loop
This is the "Smart" part of the project. In `src/predict.py`, the user can provide feedback. If the AI makes a mistake, the user corrects it, and the script appends that specific case to the `spam.csv` file. This allows the model to learn from its unique failures every time the pipeline is re-run.

---

## 🛠 Project Structure
- `main.py`: The entry point. It manages the flow between training and predicting.
- `src/`: Contains the modular scripts for Preprocessing, Training, and Prediction.
- `data/`: Stores the raw CSV and the cleaned version generated by the script.
- `models/`: Stores `spam_model.pkl` and `vectorizer.pkl` so you don't have to retrain every time you want to predict.
